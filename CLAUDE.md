### 3-3. 採用要件の実装方針サマリ

- **A-5 実環境スモーク**: デプロイ後に「過去対話のサンプル入力 → わたなべの実応答」を期待値表に持ち、クロ応答と照合する。
- **B-3 / B-6 ログ**: Python の `logging` モジュール、`LOG_LEVEL` 環境変数。ログにユーザー入力本体をフル出力しない（先頭80字 + ハッシュ）。
- **C-2 シークレット**: `.env` は **絶対に公開リポジトリへ push しない**。`.gitignore` で保護済み。Cloud Run デプロイ時はシークレットマネージャ経由。
- **E-1 再現性**: ADK Agent の生成設定で判定用パスは `temperature=0`。クリエイティブな壁打ちパスは別途。
- **E-2 出力検証**: 構造化出力には pydantic スキーマ。スキーマ不一致は黙って通さず例外で終端しログに `error_reason` を残す。

## 4. ハッカソン固有の運用ルール

### 4-1. 必ず守ること

- **Vertex AI 経由**: `GOOGLE_GENAI_USE_VERTEXAI=TRUE` を維持。Google AI Studio の API キー直叩きにしない（Free Credit 対象外になる）。
- **シークレットを公開リポジトリへ commit しない**: `.env` / サービスアカウントキー / 個人対話ログ。push 前に `git status` で必ず確認。
- **過去対話ログ（`data/raw/` 配下）は git 管理外**: `.gitignore` で保護。学習に使うがプライバシー資産として扱う。
- **コスト意識**: Vertex AI Vector Search のインデックスや Firestore は使った分だけ課金される。Free Credit $300 から消費。寝かせるインデックスは作らない／使い終わったら止める。
- **Windows 環境の日本語 stdio**: スクリプト冒頭で `sys.stdin/stdout/stderr.reconfigure(encoding="utf-8")`（→ [lessons.md L-001](lessons.md)）。

### 4-2. やらないこと（明示）

- 本格的な認証・RBAC・MFA（C-3〜C-5）。ハッカソン提出デモは個別アクセス制限で運用。
- J-SOX/ITGC 仕様書作成（D-1）。
- 障害報告書テンプレ（D-2）。
- フル冗長化・自動復旧（A-1, A-2）。Cloud Run 既定動作で十分。
- multi-pass 統合（E-3）。プロトタイプ段階では過剰投資。
